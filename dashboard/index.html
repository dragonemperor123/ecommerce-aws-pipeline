How It Works
  • 1
    Event Ingestion A Python generator replays 100k+ real Brazilian Olist orders, clickstream events, and inventory updates into three SQS queues at configurable throughput.
  • 2
    Stream Processing Lambda functions consume queue messages: the Order Processor writes to DynamoDB, the Fraud Detector flags anomalous payments via SNS, and the Inventory Alerter triggers low-stock notifications.
  • 3
    Data Lake Raw and enriched events land in S3 (raw / processed / curated zones). AWS Glue crawlers catalog the data; Athena enables ad-hoc SQL analytics over the full order history.
  • 4
    ML Recommendations A SageMaker model is trained on the Olist catalog. The Recommendation API (above) serves personalised product suggestions in real time via API Gateway + Lambda, with a DynamoDB catalog fallback.
